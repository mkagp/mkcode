# Licensing and attribution note

MK Code is derived from T3 Code. The repository's root `LICENSE` contains the
original MIT license, including the T3 Tools Inc. copyright and permission
notice. That notice remains part of copies or substantial portions of the
derived software.

MK Code additions may have separate ownership statements, but those statements
must not replace or obscure inherited license obligations. Applicable
component-specific licenses, vendored notices, embedded source notices, font
notices, and other third-party attribution remain discoverable with retained
code and distribution artifacts.

Removing an application in a future phase does not automatically make its
associated notices safe to delete. Before removing a notice, trace whether the
licensed source, binary, asset, font, generated output, substantial copy, or
distribution artifact remains anywhere in the repository or product.

Phase 1 intentionally leaves these evidence sources unchanged:

- `LICENSE`;
- `apps/web/THIRD_PARTY_NOTICES.md`;
- `apps/mobile/modules/t3-markdown-text/LICENSE` and `UPSTREAM.md`;
- `apps/mobile/modules/t3-composer-editor/LICENSE`;
- `apps/mobile/modules/t3-terminal/THIRD_PARTY_NOTICES.md`;
- `apps/mobile/modules/t3-terminal/Vendor/libghostty-vt/LICENSE`; and
- the embedded Project Nayuki MIT notice in `packages/shared/src/qrCode.ts`.

Repository evidence is not sufficient for a definitive legal conclusion about
all future distributions. In particular, mobile redistribution still needs a
targeted review of the complete license/notice bundle for MesloLGS NF fonts and
the iOS Ghostty framework. See `OPEN_QUESTIONS.md`.
